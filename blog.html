Questions and Answers

1. What is DOM and what is the purpose of DOM?

The Document Object Model (DOM) is a programming interface that provides a structured representation of web documents as a tree-like object model. It allows developers to access and manipulate web page elements using programming languages such as JavaScript, and is an essential part of web development.

There are three types of DOM:
1. Core DOM,
2. XML DOM,
3. HTML DOM

The HTML DOM treats all HTML elements as objects and views an HTML document as a tree of nodes, where each node represents an HTML element. With the HTML DOM, developers can dynamically create, change, or remove elements from a web page, as well as add events to elements like "click" or "mouseover" for interactivity.

2. How will you select HTML elements using DOM? Name the DOM methods.

There are 5 ways in which we can select elements in a DOM using selectors.

  1. getElementById()
    • This method allows us to select a specific element by its unique ID.
  2. getElementsByClassName()
    • This method allows us to select all elements that have a specific class name and returns an HTMLCollection.
  3. getElementsByTagName()
    • This method allows us to select all elements with a specified tag name and returns an HTMLCollection.
  4. querySelector()
    • This method allows us to select an HTML element using a CSS selector.
  5. querySelectorAll()
    • This method allows us to select all HTML elements that match a CSS selector and returns a NodeList.

3. What is event bubble?

Event bubbling is a concept in the Document Object Model (DOM) that occurs when an element receives an event, and that event is bubbled (or we can say propagated or broadcast) up through its ancestors and predecessors in the DOM tree until it reaches the root element.

Applying stopPropagation() method we can stop event bubbling.

4. What is a callback function and why will you use it?

A callback function is a function that is passed as an argument to another function and then executed inside the outer function to perform a specific action or routine.

We use callback functions in JavaScript to enable asynchronous programming, handle user input, make network requests, and execute code after a certain task has completed. Callbacks promote modular and reusable code, keeping code organized and easier to maintain.
